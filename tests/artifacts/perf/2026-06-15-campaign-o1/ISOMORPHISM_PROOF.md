# O1 Isomorphism Proof

Change: `OracleMcpServer` now keeps lazy `OnceLock` caches for the tool
descriptor vector and the `tools/list` result object.

Behavior contract:

- `ToolRegistry` is moved into the server at construction and is not mutated by
  any public server method.
- `oracle_capabilities` remains first in `tools/list`.
- A registry entry named `oracle_capabilities` is still deduplicated.
- Tool names, descriptions, and `inputSchema` values are generated by the same
  `descriptor_input_schema` helper as before.
- `tools/call`, session auth, OAuth scope context, and database dispatch do not
  read or write the cache.

Proof commands:

```bash
cargo fmt --all -- --check
cargo test -p oraclemcp-core server::tests::tools
```

Proof results:

- `cargo fmt --all -- --check`: passed.
- `cargo test -p oraclemcp-core server::tests::tools`: passed, including
  `tools_list_result_matches_advertised_tools_on_repeated_calls`.
- One-shot stdio initialize + `tools/list` output stayed byte-identical:
  `cmp` exit `0` for `raw/s1-before2-sample-output.jsonl` vs
  `raw/s1-after-sample-output.jsonl`.
- Both sample outputs are 44,095 bytes.

The cache stores only public tool metadata. It does not store credentials,
connection strings, query results, bind values, or auth tokens.
